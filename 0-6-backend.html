Главное про API

API — это контракт. Договор о том, как необходимо «общаться» с конкретным модулем или микросервисом. Одно конкретное взаимодействие называется API-ручкой.

Видео: что такое API — объяснение простыми словами

⚡ Знакомство со Swagger

Swagger — инструмент документирования и тестирования API. Посмотри как выглядит интерфейс Swagger Petstore — тестового магазина питомцев, с которым будем работать в ДЗ:

Swagger Petstore — видны 4 API-ручки: GET, POST, DELETE и загрузка фото

Видео: тестирование API в Swagger

🖱️ Как работать с API-ручкой — пошагово

Разберём на примере ручки POST /pet — добавить нового питомца.

Шаг 1. Найди ручку в Swagger, нажми Try it out:

Кнопка Try it out активирует режим редактирования запроса

Шаг 2–3. Замени значения в теле запроса и нажми Execute:

Меняем id: 0 → 555, name: "doggie" → "Test", нажимаем Execute

Шаг 4. Проверяем Server Response — код 200 и правильные значения в ответе:

Код 200 и значения id: 555, name: "Test" в ответе — всё прошло успешно

💡
Коды ответов HTTP

200 — успешно · 201 — создано · 400 — плохой запрос · 404 — не найдено · 500 — ошибка сервера
