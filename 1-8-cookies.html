Cookie, хранилища

Cookie, Local Storage, Session Storage — три типа хранилищ на стороне браузера. Тестировщик работает с ними в DevTools → Application.

🍪 Cookie (Куки)

Cookie — небольшие текстовые файлы, сгенерированные сайтами. Хранятся на компьютере пользователя и нужны для его идентификации.

Временные
Используются только в момент пользования сайтом. Например, данные форм заказов или просмотренные страницы. Удаляются после закрытия браузера.
Постоянные
Хранятся продолжительный период (обычно год). Содержат пароли, логины, платёжные реквизиты. Срок устанавливается сервером через response headers.
Сторонние
Принадлежат не сайту, а сторонним организациям (рекламные компании, Яндекс.Метрика, Google Analytics). Фиксируют переходы и предпочтения.

📦 Local Storage и Session Storage

Local Storage
Постоянное хранилище в браузере. Используется для сохранения пользовательских настроек: цвет темы, язык, настройки виджетов. Не очищается при закрытии браузера.
Session Storage
Временное хранилище. Часто используется для сохранения данных из полей формы. Очищается после закрытия браузера (окончания сессии).
💡
Для чего это тестеру

В следующем модуле будешь изучать как работает авторизация. По сути авторизация «живёт» в куках, реже в Session Storage. Local Storage используется для пользовательских настроек. Всё это видно в DevTools → Application.
