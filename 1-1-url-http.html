URL, протоколы HTTP и HTTPS

Разбираемся со структурой URL и протоколами которые используются в вебе каждый день.

🔗 Структура URL

URL (Uniform Resource Locator) — адрес ресурса в сети Интернет. То, что ты видишь в адресной строке браузера.

Компоненты URL: протокол, домен, путь, запрос, фрагмент

Протокол
Способ «общения» с ресурсом. Для веба это http или https.
Домен / Хост
Имя ресурса. Бывает многоуровневым. Например: divanqa.skillspace.ru — где divanqa — 3-й уровень, skillspace — 2-й, ru — 1-й (верхний).
Путь
Адрес ресурса на сервере. Зависит от структуры на бэке. Например: /catalog/toys
Запрос / Query
Набор правил для фильтрации. Всегда идёт после ?, несколько параметров разделяются &. Пример: ?color=red&pet_type=dog
Фрагмент / Hash / Якорь
Адресует к определённой части страницы. Идёт после #. Запрос на сервер не уходит — браузер сам находит якорь.

🔐 HTTP vs HTTPS

HTTP
HyperText Transfer Protocol
Протокол передачи гипертекста. С его помощью общается подавляющее большинство веб-ресурсов.
⚠️ Данные передаются открыто
HTTPS
HTTP + Secure
Расширение HTTP с шифрованием данных. Буква S означает Secure — защищённость соединения через SSL/TLS.
🔒 Данные зашифрованы
💡
SSL/TLS и TLS Handshake

SSL/TLS — технологии обеспечения безопасности соединения HTTPS. При подключении к защищённому ресурсу происходит TLS Handshake (рукопожатие) — процедура подтверждения безопасности соединения.
