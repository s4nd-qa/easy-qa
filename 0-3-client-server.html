🔗 Три компонента клиент-серверной архитектуры

💻КлиентБраузер / Приложение
запрос →→← ответ
⚙️СерверЛогика / API
SQL →→← данные
🗄️База данныхCRUD-хранилище
Дополнительно: фронтенд и бэкенд — в чём разница
Клиент
Любая точка входа в систему (браузер, мобильное приложение, десктоп-клиент). Отправляет запросы на сервер.
Сервер
Место обработки запросов в соответствии с заданной логикой. При необходимости запрашивает данные из БД.
База данных
Место хранения данных. Принцип CRUD: Create, Read, Update, Delete. Своей логики не имеет.

⚡ Фронтенд vs Бэкенд

Фронтенд
Client side
Визуальное интерактивное представление данных. То, с чем взаимодействует пользователь.
≈ то же самое что клиент
Бэкенд
Server side
Скрытая часть продукта. Прописанная логика, всё что работает «под капотом».
≈ связка сервер + БД
Видео: клиент-серверная архитектура — главное
