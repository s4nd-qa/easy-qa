Тестирование фронтенда

Заходим в браузер, открываем сайт — и тестируем. Разбираемся из чего состоит сайт и что именно проверяет фронтенд-тестировщик.

🧱 Из чего состоит сайт

Три слоя любого сайта: HTML — каркас, CSS — внешний вид, JS — поведение

HTML
HyperText Markup Language — язык разметки. Отвечает за структуру и содержание: текст, изображения, ссылки, кнопки.
CSS
Cascading Style Sheets — каскадные таблицы стилей. Отвечает за внешний вид: расположение элементов, шрифты, цвета, отступы, анимации.
JS
JavaScript — язык программирования. Пишет скрипты для фронта: интерактивность кнопок, валидация полей, динамические элементы.

🔍 Что проверяет фронтенд-тестировщик

  • Тестировать работу сайта по разным сценариям — функциональное тестирование
  • Проверять структуру и наполнение сайта, разметку HTML
  • Тестировать дизайн — сравнивать с макетами в Figma — UI-тестирование
  • Находить проблемы в работе JS-скриптов
💡
Функциональное vs Нефункциональное

Любое тестирование фронта приоритетно начинается с функционального — сначала убедись, выполняет ли система свои функции. Уже потом идёт нефункциональное (UI, адаптивность, производительность).

⚠️
Хорошего UI-тестера видно по насмотрённости

Чем чаще и скрупулёзнее сравниваешь — тем быстрее находишь неявные ошибки в разметке и стилизации. На этом сделаем упор в модуле «Фронтенд».
