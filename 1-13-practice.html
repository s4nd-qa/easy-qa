Практика по фронтенду

Большая практическая работа по фронтенду. Главная задача — научиться тестить фронт наподобие того, как это делается на реальном проекте.

Рекомендация

Убедись что все предыдущие ДЗ со статусом «принято». Если есть непонимание какой-то темы — разберись и задай вопросы в чате.

📋 Описание задачи

Представь, что фронтендер выкатил тебе на тестовый стенд новый функционал — форму обратной связи по вакансиям (фича с интеграцией Google Meet НЕ относится к задаче, её не тестировать).

📄 Технические требования (ТЗ)

Радиокнопки
Две радиокнопки для выбора режима. Названия задаются в админке, на тесте обе захардкожены как «Sales Representative».
Плейсхолдеры
Остаются после ввода — анимацией перемещаются наверх.
Name
Максимальная длина 100 символов, только буквы.
Email / Phone
Стандартная валидация.
Message
Максимальная длина 300 символов, нельзя вставлять ссылки.
Attach file
Максимальный размер 2Мб, форматы: .pdf .doc .docx. Иконка справа, при наведении — тултип «Maximum size 2MB».
Send Message
Анимация с жёлтым выделением при наведении курсора.
Валидация
Только на фронтенде (JS), невалидные запросы не отправляются на сервер. Единый стиль: тултип с описанием ошибки справа от поля — красный текст + иконка. Линия поля окрашивается в красный.
Адаптив
Поддерживает ширину экрана от 270px.
Session Storage
Данные из полей сохраняются в Session Storage до момента отправки формы.

ДЗ 1-13
1
Функциональное тестирование формы
Провести функциональное тестирование полей ввода и кнопки «Send Message». Изучи ТЗ, сначала вводи корректные данные, затем «ломай» систему. Смотри запросы в DevTools → Network. Не трать больше 2 часов.
📎 Список багов: название / ожидаемый результат / фактический результат
2
UI-тестирование
Сравнить реализованный фронтенд с макетом в Figma. Использовать PerfectPixel (масштаб картинки 0.5). Затем в DevTools → Elements проверить HTML и CSS, отступы. Не трать больше 1 часа.
📎 Список багов: название / ожидаемый результат / фактический результат
3
Кросс-браузерность и адаптив
Проверить корректность отображения на Chrome, Safari, Firefox и разных разрешениях. Не трать больше 30 минут.
📎 Список багов: название / ожидаемый результат / фактический результат
4
Восстановление после сбоев
Проверить восстанавливаются ли данные в полях после перезагрузки страницы. Согласно ТЗ, данные должны сохраняться в Session Storage. Опиши как дело обстоит на самом деле. Не трать больше 30 минут.
